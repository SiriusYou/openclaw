# Structured Brainstorming

## Process

1. **Diverge** (generate 5+ options, no filtering)
   - Think from different angles: audience, channel, timing, format, budget
   - Include at least one unconventional option
   - No evaluation yet — quantity over quality

2. **Evaluate** (score each option)
   - Feasibility (1-5): Can we execute this with current resources?
   - Impact (1-5): Expected effect on target KPI?
   - Risk (1-5, lower is better): What could go wrong?

3. **Converge** (select top 2-3)
   - Rank by (Impact × Feasibility) / Risk
   - Check for synergies between top options
   - Present to user with trade-off analysis

4. **Document** (save to memory)
   - Record all options considered (not just winner)
   - Note why rejected options were dropped
   - Tag: `memory_search("brainstorm [topic]")` for future reference
